git协议和https协议

2021-07-19

1、从命令git submodule init && git submodule update说起

  在内网服务器使用git submodule init && git submodule update命令更新仓库的子模块的时候，一直报错如下：

试了下git clone https://xxxxxx 命令可以正常下载，而git clone git@xxxxx无法正常使用，意识到问题的原因是服务器配置过http和https代理，而git协议使用的ssh并没有配置代理，接着配置ssh的proxy。

2、配置ssh的proxy

  

  • 首先使用nc命令配置，意识到代理服务器开启了密钥验证功能，所以nc命令无法正常起作用
  • 使用nc命令的升级版ncat命令进行配置，ubuntu一般没有自带ncat命令，使用apt-get install ncat命令下载安装报错，ncat无法直接安装；可以使用apt-get install nmap命令，会附带安装ncat;使用ncat命令配置之后，发现还是报错；
  • 后面又尝试了使用Corkkscrew来使用http代理，下载安装配置完成之后，也依然不行； 最后发现问题是代理服务器不支持git协议的代理，因此需要另求它法。

3、修改git submodule的连接协议

  • 简单的话直接修改文件夹下.gitmodule文件里的内容，把各子模块依赖的URL改成https协议的；有个细节是，在修改完.gitmodule文件，需要使用git submodule sync命令使修改生效，如果直接git submodule init && git submodule update依然还是无法生效；
  • 修改git的配置，强制要求使用https协议而不是git协议
